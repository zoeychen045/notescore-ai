# NotePilot for XHS

一个面向小红书发布前内容决策的 AI 产品 demo，聚焦内容质量诊断、风险识别、改写建议和反馈闭环。

## 产品定位

NotePilot for XHS 不是批量文案生成器，而是帮助创作者、品牌内容运营和 MCN / 代理商在发布前判断：

- 这篇笔记是否值得直接发布
- 当前内容的主要优势和风险是什么
- 最值得先改哪里
- 改写后是否更接近可发布状态

工具基于内容质量信号进行辅助判断，不代表小红书真实推荐机制或平台内部规则。

## 核心能力

- **发布前诊断**：根据标题、正文、内容品类、优化目标和使用者角色，输出发布建议与综合评分。
- **五维评分**：围绕首屏吸引力、真实感、信息密度、互动潜力和决策辅助进行评估。
- **风险优先级**：把内容问题转化为可执行的下一步动作，降低修改成本。
- **AI 改写稿**：基于原文已有信息优化表达，不新增未经提供的事实、价格、功效或承诺。
- **多版本对比**：对比原稿、AI 改写稿和手动版本，帮助选择更适合发布目标的版本。
- **反馈闭环**：记录用户是否采纳建议，为后续校准评分和建议质量提供样本。

## 产品价值

- **用户价值**：降低发布前判断成本，帮助创作者知道内容能不能发、应该先改哪里。
- **商业价值**：帮助品牌内容运营减少硬广感和返工，提高种草内容质检效率。
- **生态价值**：鼓励更真实、具体、有边界的内容表达，让商业内容更原生，也提升社区内容质量。

## 运行方式

1. 安装依赖：

```bash
pip install -r requirements.txt
```

2. 配置 API Key：

本地运行时，可以在 `.streamlit/secrets.toml` 中配置：

```toml
DEEPSEEK_API_KEY = "your_api_key_here"
```

也可以通过环境变量配置：

```bash
export DEEPSEEK_API_KEY="your_api_key_here"
```

3. 启动应用：

```bash
streamlit run app.py
```

## 部署注意事项

- 不要提交 `.streamlit/secrets.toml` 或任何 API Key。
- GitHub 仓库只需要保留 `app.py`、`requirements.txt`、`README.md` 等必要文件。
- 部署到 Streamlit Community Cloud 时，在平台的 Secrets 设置中添加 `DEEPSEEK_API_KEY`。

## 能力边界

- 不模拟平台真实推荐算法。
- 不鼓励夸张表达、绝对化承诺或硬广式种草。
- 不补充原文没有提供的价格、功效、数据、购买入口或其他事实。
